    Markdown学习笔记，更新后算是个人习惯写法，使用hexo-randerer-markdown-it渲染。

    # 标题

    1. 用=表示一级标题，或-表示二级标题。这个方式不常见，因为这些符号还有其他用法。

      一级标题
===

二级标题
---

二级标题
---

      一级标题

      # 二级标题

      # 二级标题

    2. 用不同数量的#标记，一个#表示一级标题，两个表示二级，以此类推。 一般使用这个方法。

      # 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

      一级

      # 二级

      # 三级

      # 四级

      # 五级
      # 六级

    # 段落

    不用特别的格式，需要换行时，句子后面加两个空格再回车，或者空出一行。 我一般使用空行换行，因为有时句子太长需要手动分成两行写， 容易看不出来后面是不是有空格。

    # 字体

    用 _ 或 * 包裹要标注的一段文字，一个 _ 或 * 表示斜体， 两个表示粗体，三个表示斜粗体。

    我一般使用*。

    *斜体*
_斜体_

**粗体**
__粗体__

***斜粗体***
___斜粗体___

    斜体 斜体

    粗体 粗体

    斜粗体 斜粗体

    # 分割线

    _，*，- 还可以用来表示分割线，需要三个或三个以上相同的符号， 中间可以有空格，但加空格干什么呢？。

    注: - 用作分割线时，上一行最好为空行，有文字的话会识别为二级标题。

    我一般使用-作为分割线。

    * * *
- - -
_ _ _

***
---
___

这个会被当作标题

    # 列表

    显示列表，分为有序与无序，列表项下的行会自动缩进，使用空行取消缩进。 列表可以嵌套。

    # 有序列表

    在一行的首位，数字加.标记，如1.，后接空格，再接列表项即可。

    注意，数字可以不按数学顺序排列，但应以数字 1 开始。谁会用乱序数字当有序列表啊。

    1. 第一项

2. 第二项，
   接在后面的文本。

3. 第三项

   空行加带缩进的文本可以对齐列表项换行，不过实际上似乎是成为了一个子项。

4. 第四项，这后面有两个空格  
这会使这行文字带缩进。很难看出来，最好少用。

    1. 第一项

    2. 第二项， 接在后面的文本。

    3. 第三项

      空行加带缩进的文本可以对齐列表项换行，不过实际上似乎是成为了一个子项。

    4. 第四项，这后面有两个空格
      这会使这行文字带缩进。很难看出来，最好少用。

    # 无序列表

    与有序列表类似，改用 * 或 - 或 + 标记，可以混用。 注: 实际显示效果可能因主题渲染不同而产生差异

    我一般使用-。

* 第二项

- 第二项

    # 代码区块

    一段代码可以使用三个反引号包裹代码段，可以在前三个反引号后指定代码语言， 这里由于渲染原因，在反引号前加了\避免样例被渲染。

    \```c
#include <stdio.h>
int main()
{
    printf("Hello world");
    return 0;
}
\```

    #include <stdio.h>
int main()
{
    printf("Hello world");
    return 0;
}

    还有一个不推荐的方法： 或者用四个空格或者制表符（TAB键）

        #include <stdio.h>
    int main()
    {
        printf("Hello world");
        return 0;
    }

    #include <stdio.h>
int main()
{
    printf("Hello world");
    return 0;
}

    # 表格

    用 | 划分列 用 - 划分表头和其他行

    可以设置对其方式: -: 整列右对齐 :- 整列左对齐 :-: 整列居中对齐

    |表头1|表头2|表头3|
|:----|----:|:---:|
| 靠左| 靠右| 居中|
| 列1 | 列2 | 列3 |
    表头1 表头2 表头3
    靠左 靠右 居中
    列1 列2 列3

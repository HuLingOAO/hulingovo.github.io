# 前言

这篇文章算是我在开发过程中的一些总结与吐槽。

其实一开始我认为对于博客来说，应当注重于文章本身，而不是花哨的样式， 这也是我一开始打算写一个自己的主题的原因。 但后面我发现，过于单调的界面很难让我阅读下去， 于是我在文章中为特殊的元素，如：链接、表格、代码等地方使用了彩色。 这确实使得文章好看了一些，也促使我为主题添加更多东西。

# 写作

主题一开始就是基于Markdown渲染出的HTML文档进行开发， 因此对于普通的，用Markdown写出来的文章，主题的样式应该都是没什么问题的。

需要特别说明的是，在Hexo的文档标签插件中提到了许多写作格式（或许可以这样子称呼？）， 如：各种不同的引用、代码块以及图片视频等。 在我一开始开发主题时并没有看中这些方面的适配，但最近在对主题进行了一系列更新期间， 我发现了官方用于主题测试的一个示例[1]， 于是我决定重新设计一些主题的的样式，并现在几乎能够适配示例中提到的各种写作风格。

除了那个diff样式，还有Hexo内置的带行号的代码高亮。 HTML标签一层又一层，真的难写，我本来就不会前端，越写越觉得写了个垃圾。

# 文章列表

主页文章列表用于展示文章所有文章，并提供访问文章的入口。 此外还有“标签页”以及“分类页”，用于展示具有相同标签或分类的文章。 注意这些页面用于展示文章，而导航栏指向的页面于展示这些标签或分类。

在官方示例中，我看到了几个我非常想要的样式：文章摘要、无标题文章、带封面文章[2]， 在主题中也实现了这些样式。

# 一般文章

在一开始设计的主页文章列表中，所有的文章都显示为一个标题加上一段文章内容。 其实我挺不满意这样的实现，因为Hexo提供的内容截取，会把标题也截进去， 更难受的是一些数学公式会因为被截断，导致不能渲染出来。

# 摘要

摘要[3]其实就是文章前的一小段文字。之前提到， 一开始我使用截取一部分文章内容的方式让我挺难受的， 于是我决定采用摘要作为一般文章的显示。

这个摘要的写法如下：

这一段文字是摘要。

<!-- more -->

这里开始是正文

在标签<!-- more -->上的内容会作为摘要显示在文章列表中，同时在文章内也会显示。

# 无标题文章

对于一篇没有标题文章，我认为它的用途就像是能在列表的一篇篇的文章中， 起到类似于文章内“引用”的作用，你可以写一小段简短的话，作为一个“引用”插入到文章列表中， 不过其实好像没什么必要。

此外，我选择实现这种布局另一个原因就是为了适配一下出现无标题文章的情况。 由于评论功能使用到了文章标题，因此在没有标题的文章里不能使用评论功能（这是强制性的）。

只需要把文章标题，也就是title设为空字符串就能使用这种样式。

# 带封面文章

一开始其实是想写个像幻灯片一样文章布局来着的， 不过我发现这些图片只能在首页显示[4]，或许更适合拿来作为文章封面。

值得一提的是，关于这个样式使用到的变量photos只是在这里提到过， 而且还没告诉你怎么用…

要为文章添加封面，可以有两种方式

  1. 在Hexo配置文件中启用post_asset_folder， 这会在创建文章时同时生成一个同名的文件夹， 把封面放入这个文件夹后，在Front matter区域添加：

    photos:
   - pic1.jpg
   - pic2.jpg
   - pic3.jpg

    这样就能使用封面了，如果有多个封面，则会随机选取。

  2. 这个还没写…主要是提供一个可供多个文章复用的封面，计划使用固定文件夹下的图片。

# 外部文章

外部文章指的是不在自己博客网站上的文章，也可以是一般的其他网页。 外部文章有两种：一种是带标题的，另一种是不带标题的。

带标题的在显示时会显示标题，而不带标题的则直接显示链接。 使用方式如下：

# 标题指定了就是带标题的，置空就是无标题的
title: ''
link: https://www.google.com

当点击时就会跳转到对应的网页，文章本身不会被访问（在考虑要不要实现访问的方式）。
